# E2E Тесты для приложения Fufelka

Проект содержит сквозные E2E-тесты для проверки авторизации и управления профилем в приложении Fufelka.

## Технологии
- Cypress 13.6.4
- JavaScript

## --- Как запустить тесты ---

### 1. Установка зависимостей
npm install

## Запуск в headless режиме
npm run test:e2e

## Запуск в интерактивном режиме (Cypress Test Runner)
npm run cypress:open

## --- Какие сценарии покрыты ---
- Проверка защиты роутов - недоступность защищенных страниц без авторизации
- Успешная авторизация - вход с валидными учетными данными через UI
- Валидация ответа API - перехват и проверка auth-эндпоинта
- UI подтверждение - проверка отображения пользовательских данных
- Нет жестких таймаутов (cy.wait(1000)
- Сохранение сессии - многократные перезагрузки с проверкой токенов
- Навигация в профиль - переход и проверка данных профиля
- Загрузка аватара - upload через input[type="file"]
- Перехват запросов - отслеживание upload и update запросов (используется intercept() + wait('@alias'))
- Валидация изменений - проверка обновления src аватара
- Сохранение после reload - подтверждение достоверности данных
- Очистка после всех тестов
- Видео прогона скинул в телеграмм, также сохраняется в папке videos (только при запуске headless режиме)
- Также в ТГ приложил скриншоты (тест длится не более 6 сек)
- В profile-auth.cy.js можно раскомментировать стр 150 ( // cy.wait(20000); ) и окно с тестами не схлопнется

## --- Какие данные генерируются автоматически ---
В файле cypress/support/commands.js:
    timestamp,                   // Текущая метка времени
    randomString,                // Случайная строка (base36)
    uniqueUsername: `testuser_${timestamp}_${randomString}`,
    uniqueEmail: `test_${timestamp}_${randomString}@test.com`
